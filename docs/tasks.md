# 金融资讯智能分析系统 - 任务拆解清单（RAG优先查询工作流）

## 1. 使用说明
- 本文档用于指导 Sprint 执行，按里程碑（M1-M5）推进。
- 每个里程碑按三条泳道拆分：后端、Agent、前端。
- 每项任务包含交付物与验收勾选，方便执行与复盘。

## 2. 全局优先级
- P0：M1 查询协议与意图结构化、M2 RAG命中判定
- P1：M3 未命中回退抓取闭环、M4 每日08:00调度
- P2：M5 可观测性与质量体系完善

## 3. 里程碑任务

## M1 查询协议与意图结构化（P0）

### 后端泳道
- [ ] 定义查询接口 DTO：`query`、`market`、`time_range_hours`、`trace_id`
  - 交付物：请求/响应 schema 文档
  - 验收：字段约束明确、默认值为 24h
- [ ] 新增查询编排入口 `POST /query/analyze`
  - 交付物：接口草案与错误码约定
  - 验收：接口可返回结构化意图占位数据

### Agent泳道
- [ ] 实现自然语言意图结构化
  - 交付物：结构化字段定义（raw_query/keywords/sector/market/time_range_hours/intent_type）
  - 验收：典型查询可稳定解析
- [ ] 增加兜底规则
  - 交付物：规则清单（识别失败回退策略）
  - 验收：无法识别时强制保留 raw_query 与 24h 默认值

### 前端泳道
- [ ] 搭建最小查询页（输入框 + 市场 + 时间范围）
  - 交付物：页面原型与字段映射说明
  - 验收：可提交查询请求
- [ ] 增加意图调试展示区
  - 交付物：结构化意图可视化区块
  - 验收：前端可展示后端返回的 intent

### M1里程碑验收
- [ ] 同一用户问题能稳定返回结构化意图
- [ ] 查询页面可完成提交并展示 intent

## M2 RAG优先检索与命中判定（P0）

### 后端泳道
- [ ] 新增检索服务层（按 `market`、`published_at` 过滤）
  - 交付物：检索服务接口与过滤条件说明
  - 验收：支持 24h/48h 时间窗过滤
- [ ] 实现命中判定器
  - 交付物：配置项文档（`similarity_threshold`、`min_docs`、`source_diversity`）
  - 验收：可输出命中/未命中决策与原因

### Agent泳道
- [ ] 将结构化意图映射为检索条件
  - 交付物：检索参数构建规则
  - 验收：关键词、板块、市场映射一致
- [ ] 命中场景输出首轮分析
  - 交付物：命中场景分析模板
  - 验收：无需抓取即可返回分析结果

### 前端泳道
- [ ] 展示证据列表（来源/时间/相关度）
  - 交付物：evidence 组件
  - 验收：至少展示来源、URL、发布时间
- [ ] 展示命中状态
  - 交付物：命中/未命中状态标签
  - 验收：状态可与后端判定一致

### M2里程碑验收
- [ ] 命中场景不触发实时抓取
- [ ] 返回证据均满足 24h/48h 时间约束

## M3 未命中回退抓取闭环（P1）

### 后端泳道
- [ ] 串联未命中回退流程（检索不足 -> 批量抓取 -> 分析 -> 入库 -> 入向量库）
  - 交付物：流程编排图与状态流转说明
  - 验收：未命中场景可自动补抓并回写
- [ ] 接入重试与审计日志
  - 交付物：审计字段映射说明（`retry_count`、`audit_log_id`）
  - 验收：单条失败不影响整批，失败可追踪

### Agent泳道
- [ ] 合并“历史命中 + 新抓取”进行二次分析
  - 交付物：融合分析模板
  - 验收：输出统一结构（summary/impact/sectors/reasoning/risk_notice）

### 前端泳道
- [ ] 增加实时抓取触发标识
  - 交付物：meta 区域展示字段
  - 验收：可显示 `fallback_triggered`
- [ ] 展示处理耗时与来源占比
  - 交付物：处理统计组件
  - 验收：可展示检索与抓取贡献比例

### M3里程碑验收
- [ ] 未命中时可自动完成回退抓取并返回结果
- [ ] 结果包含重试次数和审计日志ID

## M4 每日08:00定时任务与看板（P1）

### 后端泳道
- [ ] 配置每日 08:00 调度任务
  - 交付物：调度配置文档与执行入口
  - 验收：可按计划触发批量执行
- [ ] 新增任务查询与补跑接口
  - 交付物：`POST /jobs/daily-run`、`GET /jobs/daily-run/{job_id}` 契约
  - 验收：可查询任务状态与结果汇总

### Agent泳道
- [ ] 设计日报分析模板
  - 交付物：每日汇总输出模板
  - 验收：可生成“今日重点事件+影响摘要”

### 前端泳道
- [ ] 新增“今日任务看板”
  - 交付物：总数/成功/失败/失败原因展示
  - 验收：可按任务ID查看详细执行结果

### M4里程碑验收
- [ ] 连续 3 天定时任务可稳定运行
- [ ] 任务失败可定位到具体 URL 与错误原因

## M5 质量体系与可观测性（P2）

### 后端泳道
- [ ] 增加核心指标埋点
  - 交付物：指标清单（命中率、回退率、耗时、失败率）
  - 验收：指标可查询、可对比
- [ ] 增加核心链路自动化测试
  - 交付物：测试用例清单（意图解析、命中判定、回退抓取）
  - 验收：核心链路回归可执行

### Agent泳道
- [ ] 增加输出质量校验
  - 交付物：字段完整性与时间窗一致性校验规则
  - 验收：输出字段完整率达标

### 前端泳道
- [ ] 完善错误态与空数据态
  - 交付物：异常展示规范
  - 验收：查询失败与无结果时有明确反馈

### M5里程碑验收
- [ ] 关键字段完整率 >= 95%
- [ ] 核心链路回归测试通过

## 4. 执行顺序建议
- 第一阶段：M1 -> M2（先建立协议和检索判定）
- 第二阶段：M3 -> M4（补全闭环与调度运营）
- 第三阶段：M5（质量与可观测性收口）

## 5. 每周例行检查项
- [ ] 本周查询命中率是否提升
- [ ] 回退抓取比例是否下降
- [ ] 失败原因 Top3 是否已处理
- [ ] 24h/48h 时效约束是否持续生效
