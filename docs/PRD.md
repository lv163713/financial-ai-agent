# 金融资讯智能分析系统 - 需求文档 (PRD)

## 1. 项目概述
本项目旨在为金融行业从业者（如分析师、交易员、研究员）提供一个智能化的资讯处理平台。系统通过自动爬取每日全球重要财经新闻与事件，并结合AI大模型分析这些事件对特定股市板块及个股的潜在影响，最终通过直观的Dashboard展示给用户，帮助用户在开盘前快速把握市场脉络。

## 2. 目标用户
- 金融机构研究员/分析师
- 个人投资者
- 财经媒体工作者

## 3. 核心功能需求

### 3.1 资讯自动采集 (Web Scraper Agent)
- **数据源获取**：自动从指定的专业财经网站抓取最新资讯。
- **反爬策略**：集成 Jina Reader API，实现稳定、高效的文章正文提取，转换为大模型易于理解的 Markdown 格式。
- **定时任务**：支持设置定时抓取（例如：每日早晨 7:30 自动执行一次批量抓取）。

### 3.2 智能分析引擎 (AI Analysis Agent)
- **接入模型**：调用 阿里云通义千问大模型 (Tongyi Qianwen) API。
- **分析维度**：
  - **事件摘要**：用一句话总结新闻核心事件。
  - **影响评估**：判断该事件是利好、利空还是中性。
  - **板块映射**：指出受该事件直接或间接影响的行业板块。
  - **逻辑推演**：简述为什么该事件会产生上述影响。

### 3.3 数据存储与管理 (Data Management)
- **数据库系统**：MySQL (使用 DataGrip 管理)。
- **资讯库**：持久化存储抓取到的原始新闻标题、链接、发布时间和内容。
- **分析库**：存储AI大模型生成的结构化分析结果（JSON格式），与原始新闻建立关联。

### 3.4 前端可视化展示 (Dashboard)
- **今日速览 (Daily Overview)**：按时间线展示过去 24 小时的核心财经事件。
- **板块热度榜 (Sector Impact)**：统计今日受利好/利空影响最大的板块排名。
- **详情卡片 (Detail Cards)**：点击单条事件，展开展示 AI 生成的内容。

## 4. 迭代计划
- **Phase 1 (MVP)**：完成单个数据源的抓取，接入通义千问进行分析，本地 MySQL 存储，本地实现简单的前后端展示。
- **Phase 2**：完善定时调度，丰富前端图表，使用 Docker 部署至个人云服务器。
